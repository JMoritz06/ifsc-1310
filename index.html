Web Accessibility

Web accessibility is very important as the internet/web grows. To me it is basically how available the web is to all parties. This includes all ages, genders, disabled persons being able to access the same pages and resources. It is very crucial to question web accessibility when creating a web page because that is the point of the web. To have anyone able to access the resources that they may need. Whether that is including code for a screen reader or having the layout simple for easy understanding. For web developement to grow web accessibility must be put on the top of the priority list.

Thoughts on Progressive Enhancement

Progressive enhancement is the web developement strategy of starting with the content and then building up. Basically starting with structered html and then inputting the content. Then after all content is added moving on to presentation like css and features like javascript. This startegy is really important for the future of web development for a few reasons. With modern technology there are so many different ways of accessign the web like through a phone or computer. It is important to allow access from all devices to advocate for anyone to vist a site. So building from the content first allows anyone to be able to access said content instead of being limited by their technology or even their browser. Progessive enhancement kind of ties to web accessibility as it allows for the developers to have an easier time allowing different windows versions or old browsers have access to their site. It really differs from graceful degradation which starts at a high and focuses on accessibility last. Progressive enhancement is a positive strategy and should be used extensively.
